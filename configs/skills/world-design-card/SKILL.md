# 世界观设计卡生成技能

## 技能说明

本技能用于从用户输入的任意格式文档中提取世界观信息，转换为结构化表格。适用于所有类型世界观。

## 在本项目中的文件约定（Discord 世界系统）

当你在本仓库的“世界构建会话”（通常由 `/world create` 进入的私密对话）中使用本技能时：

- 必须先读取：`world/source.md`（创作者上传的设定原文）
- 必须写入：`world/world-card.md`（世界卡）与 `world/rules.md`（底层规则）
- 注意：只有在“世界构建私密对话”（创作者会话）写入的文件才会被持久化；在游玩会话里它们是只读的，写了也不会保存
- 必须通过工具写入/编辑文件，禁止只在聊天里输出最终内容
- 只输出必要模块：仅包含能从 `world/source.md` 与对话中确认的模块，未提及模块直接省略
- 信息不足时不要用“无”填满模板；优先写核心模块并列出待补充清单
- 控制输出长度，避免无意义的超长回复
- 回复中只做：变更摘要 +（如信息不足）3-5 个需要创作者补充的问题；如果信息已足够则明确说明“已 OK”，并提醒创作者执行 `/world publish`

约束建议：

- `world/world-card.md`：只放结构化“世界观正典信息”（名称、背景、势力、地点、历史等）
- `world/rules.md`：只放“底层硬规则”（初始金额/装备、物理/超自然规则、禁止事项等）

| 战争类 | 战锤、军事、战国、三国、星际战争            |
| 游戏类 | 宝可梦、游戏王、RPG、MMORPG、系统流、无限流 |
| 魂系类 | 黑魂、艾尔登法环、死亡循环、Roguelike       |
| 表里类 | Persona、都市怪谈、双层世界                 |

## 使用规则

1. 所有模块均为可选，根据用户内容决定填充
2. 用户未提及的模块直接省略（不要用“无/N/A”填满模板）
3. 支持用户自定义字段
4. 特殊机制模块按需选用
