# ADR-006: 平台优先级调整（先 Discord，QQ 延后）

## 1. 背景 (Context)

现阶段同时维护 QQ（llbot/国密协议栈）和 Discord，两端都需要适配、测试与合规。当前开发人力有限，且 QQ 端协议与封禁风险更高，阻塞了交付节奏。

## 2. 决策 (Decision)

- **第一优先**：Discord。功能、接口与测试以 Discord 为基线先落地、上线。
- **延后/可选**：QQ。QQ 相关实现/验证不再卡住主线，可择机补齐或暂缓。

## 3. 理由 (Rationale)

- 风险：QQ 协议栈变动频繁、封号/风控成本高，环境搭建复杂。
- 价值：Discord 生态稳定、外部依赖简单，能更快交付真实用户价值。
- 成本：集中精力在一个平台上可显著降低测试矩阵与运维负担。

## 4. 影响 (Consequences)

- PR/发布优先保证 Discord 端可用；QQ 问题不阻断主线合入。
- 新功能/脚本默认以 Discord 为兼容基线；QQ 需额外时间评估。
- 测试矩阵：先跑/修 Discord，用特性标记或跳过 QQ 测试。
- 文档与配置：默认样例/指南以 Discord 为主，QQ 配置可放在附录。

## 5. 未来工作 (Follow-ups)

- 为 QQ 端补充单独的兼容性清单与风险提示（封禁、登录、网络要求）。
- 当人力/需求到位时，再按清单逐项恢复/验证 QQ 功能。
